Addressing these questions:

> Can a clear functional and technical specification be given of federated SPEs based on data stations, consistent with existing reference architectures?
> In what way could data stations be standardised to achieve maximum interoperability of federated SPEs?

Preliminary conclusions:

- Unity of technology is already possible to achieve technical interoperability
    - A data station is in fact a 'mini-SPE' / decentralised SPE and must therefore meet all EHDS requirements
    - This is achievable, but there are a few gaps (see [proposed development agenda](ontwikkelagenda.md))
- Unity of language will take longer (semantic interoperability) --> within data stations, maximally leverage the composable data stack to efficiently and effectively support data transformations
- A large part of the TEHDAS2 technical requirements are already realised in KIK-V and PLUGIN.
